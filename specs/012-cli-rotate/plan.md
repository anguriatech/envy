# Implementation Plan: Envelope Passphrase Rotation

**Branch**: `012-cli-rotate` | **Date**: 2026-06-10 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/012-cli-rotate/spec.md`

## Summary

Add a dedicated `envy rotate` CLI subcommand that re-seals an existing envelope in `envy.enc` with a new passphrase, after verifying the current passphrase matches the existing envelope. This fills the safety gap left by `envy encrypt`'s silent key-rotation behaviour in headless mode (used by the VS Code extension, where a typo silently changes the envelope's passphrase and locks the team out).

The implementation is purely additive: no existing subcommand is modified, no new crate dependency is added, and no new error variant is introduced. The new `cmd_rotate` lives in `src/cli/commands.rs`; the verify-then-seal logic lives in `src/core/sync.rs` as a new `rotate_env` helper, parallel to the existing `seal_env`.

## Technical Context

**Language/Version**: Rust stable (edition 2024, MSRV 1.85)
**Primary Dependencies**: `clap` (derive), `dialoguer` (Password + MultiSelect), `serde_json`, `zeroize`, `thiserror` — all already in `Cargo.toml`. No new crate added.
**Storage**: SQLite via `rusqlite` with `bundled-sqlcipher-vendored-openssl` (existing vault, read-only for this feature). `envy.enc` artifact on disk (JSON, atomic write via existing helper).
**Testing**: `cargo test` for unit and integration tests; `tests/e2e_devops_scenarios.sh` for E2E. Existing `TEST_MASTER_KEY` + `ENV_LOCK` pattern in `src/cli/commands.rs:1540` is reused — tests do NOT need an OS keyring.
**Target Platform**: Linux + macOS + Windows (already supported by envy; rotation inherits the same platform support).
**Project Type**: CLI tool (single binary, statically linked).
**Performance Goals**: < 30 seconds for a single-env rotation in interactive mode (SC-001). Rotation is dominated by Argon2id KDF cost (~hundreds of milliseconds) plus JSON I/O; no special performance work needed.
**Constraints**: Atomic write of `envy.enc` (FR-016); zero plaintext passphrase in memory (Constitution Principle I); existing 4-layer separation (Constitution Principle IV); no new crate deps; no new error variants.
**Scale/Scope**: One new subcommand, one new core helper, ~9 unit tests + 1 E2E scenario. No scope expansion to revocation, audit log, scheduling, or extension.

## Key Files & Functions

### `src/core/sync.rs` — new `rotate_env` function

```rust
pub fn rotate_env(
    vault: &Vault,
    master_key: &[u8; 32],
    project_id: &ProjectId,
    artifact: &mut SyncArtifact,
    env_name: &str,
    current_passphrase: &str,
    new_passphrase: &str,
) -> Result<(), SyncError>
```

The function:
1. Validates `new_passphrase` is non-empty/non-whitespace → `SyncError::Artifact(ArtifactError::WeakPassphrase)`.
2. Looks up `artifact.environments[env_name]` → `SyncError::Artifact(ArtifactError::MalformedArtifact("env not in artifact"))` if missing.
3. Verifies `current_passphrase` via `check_envelope_passphrase(...)` → `SyncError::Artifact(ArtifactError::DecryptionFailed)` if false.
4. Calls `seal_env(vault, master_key, project_id, env_name, new_passphrase)` to produce the new envelope (re-uses the existing seal + sync_marker write logic).
5. Inserts the new envelope into `artifact.environments[env_name]`, replacing the old one.
6. Returns `Ok(())`.

### `src/cli/commands.rs` — new `cmd_rotate` function

```rust
pub(super) fn cmd_rotate(
    vault: &Vault,
    master_key: &[u8; 32],
    project_id: &ProjectId,
    artifact_path: &std::path::Path,
    env_filter: Option<&str>,
) -> Result<(), CliError>
```

The function:
1. Reads the artifact via `crate::core::read_artifact(artifact_path)` to get the list of envs for MultiSelect.
2. Determines the envs to rotate: single env from `env_filter`, or MultiSelect from the artifact.
3. For each selected env, in order:
   - Empty-env guard (mirror `cmd_encrypt` at lines 727-736).
   - Resolve current passphrase via `resolve_passphrase_for_env(env_name, false, None)`. Wrap in `Zeroizing<String>`.
   - Resolve new passphrase (interactive via `dialoguer::Password::with_confirmation`, or headless via `ENVY_PASSPHRASE_<ENV>_NEW`). Wrap in `Zeroizing<String>`.
   - Reject `new == current` (compare as `&str`).
   - Call `crate::core::rotate_env(...)`.
   - Print the success line (✓ + env name + forward-only note).
4. Calls `crate::core::write_artifact_atomic(&artifact, artifact_path)`.

**Helper**: `is_rotate_headless_mode(env_name) -> bool` — returns `true` only when BOTH `ENVY_PASSPHRASE_<ENV>` and `ENVY_PASSPHRASE_<ENV>_NEW` are set and non-empty. The existing `is_headless_mode` helper is NOT reused because it does not check the `_NEW` variant.

### `src/cli/mod.rs` — new `Commands::Rotate` variant

```rust
/// Re-seal an existing envelope with a new passphrase.
///
/// The current passphrase is verified before the new one is accepted.
/// Prompts for the current, new, and confirmation passphrases interactively,
/// or reads `ENVY_PASSPHRASE_<ENV>` and `ENVY_PASSPHRASE_<ENV>_NEW` in CI.
Rotate {
    /// Target environment to rotate (default: MultiSelect from envy.enc).
    #[arg(short = 'e', long = "env", value_name = "ENV")]
    env: Option<String>,
},
```

Plus a dispatch arm in `run()` that calls `commands::cmd_rotate(...)`.

## Test Strategy

### Unit tests in `src/cli/commands.rs` (7 new tests)

1. `rotate_happy_path_seals_then_unseals_with_new` — full round-trip: seal with A, rotate to B, decrypt with B succeeds, decrypt with A fails.
2. `rotate_wrong_current_passphrase_leaves_artifact_unchanged` — SHA-256 of `envy.enc` before and after a wrong-passphrase attempt is identical.
3. `rotate_new_equals_current_is_rejected` — error returned, artifact SHA-256 unchanged.
4. `rotate_confirmation_mismatch_is_rejected` — error returned, artifact SHA-256 unchanged.
5. `rotate_headless_with_env_vars_succeeds` — uses `ENVY_PASSPHRASE_<ENV>` + `ENVY_PASSPHRASE_<ENV>_NEW`, no TTY.
6. `rotate_multi_environment_rotates_each` — MultiSelect with two envs, both re-sealed.
7. `rotate_empty_env_skips_with_warning` — vault has 0 secrets, warning printed, artifact unchanged.

### Unit tests in `src/core/sync.rs` (1-2 new tests for `rotate_env`)

1. `rotate_env_verifies_current_passphrase_before_sealing` — wrong current passphrase returns `DecryptionFailed`, artifact is unchanged.
2. `rotate_env_replaces_envelope_in_artifact` — happy path: new envelope is byte-different from old, other envelopes unchanged.

### E2E scenario in `tests/e2e_devops_scenarios.sh` (Scenario 10)

- `envy init`, `envy set KEY=val`, `envy encrypt -e development` with passphrase A, `envy rotate -e development` with A→B, `envy encrypt -e development` with B, `envy decrypt` with A (development skipped, exit 1), `envy decrypt` with B (development imported). Plus a wrong-current-passphrase path that asserts `envy.enc` SHA-256 unchanged.
